# 追加修正: ダメージ表記（非攻撃時の非表示 / ラベル削除）

前提: 命中バッジ・内訳ダメージ（発生源 SP/本体）は実装済み。以下2点のみ。

## 1. 攻撃を受けていない側はダメージ表記を出さない
判定は**被命中数**で行う（ダメージ値ではない）。命中数0＝攻撃を受けていない→非表示。命中あり・ダメージ0は「0」表示を維持。

- SP表記（上部バースト）: その unit の被SP命中数 > 0 のときのみ表示。
  被SP命中数 = unit が守備側なら `atkToMeme + atkSelfMemeHits`、攻撃側なら `defToMeme + defSelfMemeHits`。
- 本体表記（下部ポートレート）: 被本体命中数 > 0 のときのみ表示。
  被本体命中数 = 守備側 `atkToChar + atkSelfCharHits`、攻撃側 `defToChar + defSelfCharHits`。
- `allyIsAttacker` で atk/def を ally/enemy にマップ（既存のダメージ配線と同方式）。

スクショ例（遠距離）: しゅおは反撃を受けず被命中0 → SP・本体とも非表示。中部つるぎは被命中あり → 表示（内訳の 本体発 −0 等は表示のまま）。

## 2. ラベル文字を消し数字だけにする
以下の文字列を削除。数値は残す。
- ヘッダ `SP DAMAGE` 削除 → 合計値の数字のみ。
- 内訳ラベル `SP発` / `本体発` 削除 → 内訳は数字のみ。発生源は**色**で区別（SP発＝STREAM_SP、本体発＝STREAM_GEN。命中バッジ色と一致）。
- 下部ポートレートのヘッダ `本体 DAMAGE` も同様に削除（「数字だけ」方針で統一）。

表示イメージ（中部つるぎ・被弾側）:
```
   −10            ← 合計(無ラベル)
   −10 (緑)       ← SP発(色のみ)
   −0  (紫)       ← 本体発(色のみ)
```

## 確認（?qa=battlefull）
- 遠距離で攻撃側が反撃を受けない場合、その側に −0 表記が一切出ない。
- 被弾側は合計＋内訳2数字（色分け）が出る。ラベル文字 `SP DAMAGE`/`本体 DAMAGE`/`SP発`/`本体発` は無い。
- 被弾あり・特定発生源0は「0」を表示（色で識別）。
